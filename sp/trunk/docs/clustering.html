OIOSAML.java Clustering

Part of the state required for OIOSAML.java to work is stored in static variables by default. This means that the implementation is not cluster safe. However, the default implementation can be changed, so it is possible to implement a safe state storage instead. The relevant interface is dk.itst.oiosaml.sp.service.session.SessionHandler. The concrete type which is used at runtime is controlled by the configuration property oiosaml-sp.sessionhandler, which holds the class name of a class implementing the SessionHandler interface.

The following SessionHandler implementations are available in the standard distribution:

SingleVMSessionHandler

This handler uses static maps to save all session state. This means that it is not suited for a distributed environment. However, it's fast and efficient, and should be used whenever OIOSAML.java is installed on a single server.
