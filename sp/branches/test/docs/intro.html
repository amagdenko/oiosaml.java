OIOSAML.java Service Provider Introduction

The OIOSAML.java Service Provider package implements a Servlet-compliant SAML Service Provider for use in a SAML federation. The package implements all necessary functionality to work as a OIOSAML compliant Service Provider.

The main component of the package is a servlet filter which handles user authentication. The filter checks if the user is already authenticated, and if not, the user is redirected to the SAML Identity Provider. When the user returns with a SAML assertion, the assertion is validated, and a new session is created for the user. The application can access the received assertion and its attributes through a programming API.

IdP and Service Provider configuration is handled through standard SAML metadata.

Contents

Contents of the OIOSAML.java distribution;

  • doc: This documentation
  • lib: Jar file dependencies for OIOSAML.java
  • src: Source files for the project
  • oiosaml.java-discovery-x.x.war: OIOSAML Discovery service. See discovery guide.
  • oiosaml.java-sp-x.jar; OIOSAML.java library. Use this together with lib/ when embedding OIOSAML.java in an existing project. See installation guide.
  • oiosaml.java-sp-demo-x.war; Demonstration project as a web application. Deploy to a servlet container to see OIOSAML.java in action. See installation guide.

Requirements

There are only a few requirements for OIOSAML.java:

  • Java 6+.
  • Java Cryptography Extension (JCE) Unlimited Strength Jurisdiction Policy Files
  • A web container, for example Tomcat supporting at least Servlet 2.4.
  • A SAML 2.0 compliant IdP. Assertions must be signed, as must SLO requests and responses.

OIOSAML.java has been tested with:

  • Java 1.6.0_45 and Tomcat 7.0.42
  • Java 1.6.0_03 and Tomcat 6.0.16
  • Java 1.6.0_03 and Trifork P4
