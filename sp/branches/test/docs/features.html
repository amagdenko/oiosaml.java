OIOSAML.java features

The main features of OIOSAML.java are:

  • SAML 2.0 compliant
  • OIOSAML compliant
  • Implements servlet filter for automated user authentication
  • Signatures are checked automatically
  • Encryption of SAML assertions can be enforced
  • Automatic CRL checking of IdP certificates
  • Discovery profile and multiple identity providers are supported
  • ForceAuthn can be set on selected urls
  • NameIDPolicy can be selected using properties
  • IsPassive can be configured. If IsPassive is set, SPFilter will allow unauthenticated requests, but the context assertion will not be valid. See developer's guide.
  • Supports AttributeQuery
  • Error page can be customized
  • Custom Assertion validators can be injected
  • POST requests are saved if they initiate SSO process and are replayed after authentication
  • Session state can be saved in a number of ways, thereby supporting both standalone and clustering
  • A special developer mode which does not require a real IdP

Supported bindings

The following SAML bindings are supported by OIOSAML.java:

  • Single Logout: HTTP Redirect
  • Single Logout: SOAP
  • Login request: HTTP Redirect
  • Login request: HTTP POST
  • Login response: HTTP Artifact
  • Login response: HTTP POST
  • Login response: HTTP Redirect

Supported Profiles

The following SAML protocols are supported by OIOSAML.java:

  • Web Single Signon
  • Single Logout
  • IdP Discovery
  • Artifact Resolution
  • OIOSAML OCES Attribute profile
  • Attribute Query

Signatures and encryption

OIOSAML.java requires that all requests and responses are signed with a XML digital signature. The signatures are automatically validated upon reception, and OIOSAML.java will automatically insert signatures in outgoing requests.

Encryption is only supported when receiving assertions, and only the <EncryptedAssertion> element is supported. This means that only whole assertions can be encrypted, encrypted attributes are not supported, and will be ignored. OIOSAML.java can be configured to allow unencrypted assertions by setting oiosaml-sp.encryption.force to false in oiosaml-j.properties.

Metadata is signed automatically when downloaded through the saml/metadata servlet.

Limitations

  • The default configuration is not cluster safe. See the clustering guide for more information.
