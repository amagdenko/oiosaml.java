WS-Federation extension for OIOSAML.java

This package contains a WS-Federation extension for OIOSAML.java, an open source SAML 2.0 Service Provider implementation for Java. The package has primarily been built for use with the Microsoft Geneva framework, but should work with other WS-Federation compliant tools.

The package works by installing a servlet filter, which handles all signon and signout protocol messages. The filter also makes sure that requests are authenticated correctly, so that application code does not have to handle this manually. Application code can then read the attributes received in the assertion in order to implement business logic.

  • Installation of the demo application
  • Installation into an existing or new web application

WS-Federation Requirements

The extension is based on the following assumptions:

  • JDK 1.5 or higher
  • No wreq parameter is sent to the STS when initiating login
  • The contents of wresult is a wst:RequestSecurityTokenResponse
  • The RequestedSecurityToken element contains either saml2:Assertion or saml2:EncryptedAssertion
  • The assertion is signed and valid according to the SAML 2.0 specification

Further Information

The WS-Federation extension is based on OIOSAML.java, which can be customized in a number of ways. Read the full documentation for OIOSAML.java to learn how to extend and configure the package.

The extension uses the same configuration mechanism as OIOSAML.java - files are placed in ~/.oiosaml-appname, and the main config file is oiosaml-sp.properties.
