Requirements for the WS-Federation Extension

Before the extension is installed into an application server, there are some configuration changes which are necessary:

Prepare the JDK for unlimited crypto

To support encryption, the JCE Unlimited Strengh crypto support must be installed. To do this, go to java.sun.com, go to download of the appropriate JDK version and select "Java Cryptography Extension (JCE) Unlimited Strength Jurisdiction Policy" (probably at the bottom). This is a small zip file, containing two jar files. These two files must be copied to JAVA_HOME/jre/lib/security, overwriting the existing files.

Endorse new Xerces version

OIOSAML.java requires Xerces 2.9.1 in order to work. Because the JDK already contains an XML parser, the old one must be overwritten. Choose one of the following options:

  • (Recommended) Copy the META-INF dir from the distribution file to WEB-INF/classes in the application. This keeps the change local to your application, and does not change anything in any other applications which might be deployed on the same server. However, this method requires a recent version of the JDK.
  • Endorse xerces in the application server. The OIOSAML documentation has information on how to do this.
  • It might also be necessary to specify the correct implementation using system properties. For example, the GlassFish container sets a property which specifies another parser, which must then be overwritten.
